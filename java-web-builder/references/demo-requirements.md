# Demo 功能需求文档

## 1. 文档信息
- 功能名称: Demo 功能
- 文档目标: 明确 Demo 查询能力的业务边界、输入输出与验收标准

## 2. 背景与目标
系统需要提供统一的 Demo 查询能力，根据名称直接查询 MySQL 数据并返回结果。

## 3. 功能需求
1. 提供名称查询能力。
2. 输入参数包含名称，名称不能为空，长度不超过 64 个字符。
3. 检索策略为直接查询 MySQL 返回结果。
4. 返回结果采用统一响应结构。
5. 当 MySQL 无数据时，返回业务失败信息。

## 4. 业务规则
1. 名称为空或空白字符时，请求视为非法。
2. 名称长度大于 64 个字符时，请求视为非法。
3. MySQL 查询不到数据时，返回失败响应。

## 5. 输入输出要求
### 5.1 输入
- 输入字段: 名称
- 取值约束: 非空，长度不超过 64 个字符

### 5.2 成功输出
- success: true
- message: 成功信息
- data: 包含查询结果

### 5.3 失败输出
- success: false
- message: 失败原因

## 6. 验收标准
1. 输入参数满足校验规则时，返回成功且结果不为空。
2. 当输入有效名称时，应直接查询 MySQL 返回结果。
3. 当 MySQL 无数据时，应返回失败和业务错误信息。
4. 参数非法（空值、超长、错误请求体）时，应返回统一失败结构。
